1. INTRODUCTION
================================================================================

1.1 Purpose
-----------
This document specifies the software requirements for the Speed Limiter System
designed to control vehicle acceleration by intercepting and modifying brake
pedal signals when the vehicle exceeds a predefined speed limit.

1.2 Scope
---------
The Speed Limiter System is an embedded control system that:
- Monitors vehicle speed via CAN bus communication
- Intercepts accelerator pedal signals when speed limit is exceeded
- Generates default pedal signals to prevent further acceleration
- Operates in two distinct modes: Manual and Automatic
- Provides visual/audible warnings to the driver

1.3 System Overview
-------------------
The system is designed for vehicles with electric wired brake pedals. When the
vehicle exceeds the configured speed limit, the system:
1. Cuts the connection between the actual pedal signals and the ECU
2. Sends default signals to the ECU to prevent acceleration commands
3. Activates warning indicators
4. Restores normal operation when speed decreases below the limit

2. FUNCTIONAL REQUIREMENTS
================================================================================

2.1 Speed Monitoring (FR-001)
------------------------------
REQ-FR-001.1: The system SHALL continuously monitor vehicle speed from CAN bus
REQ-FR-001.2: The system SHALL support speed limit configuration (default: 40 km/h)
REQ-FR-001.3: The system SHALL calibrate CAN speed readings to match dashboard
              speed (target: dashboard speed limit 40 km/h)
REQ-FR-001.4: The system SHALL update speed readings at a minimum frequency of
              10 Hz

2.2 Speed Limit Enforcement (FR-002)
--------------------------------------
REQ-FR-002.1: When vehicle speed >= speed limit, the system SHALL:
              - Activate relay to cut ECU signals (HIGH state)
              - Generate and send default pedal signals to ECU
              - Activate buzzer warning tone
              
REQ-FR-002.2: When vehicle speed < speed limit, the system SHALL:
              - Deactivate relay to reconnect pedal signals (LOW state)
              - Stop sending default signals
              - Deactivate buzzer warning tone

2.3 Operating Modes (FR-003)
-----------------------------
REQ-FR-003.1: The system SHALL support two operating modes:
              - Mode 1: Manual (potentiometer-based)
              - Mode 2: Automatic (ESP32-based)

REQ-FR-003.2: Mode selection SHALL be determined by hardware switch state

2.4 Mode 1 - Manual Operation (FR-004)
---------------------------------------
REQ-FR-004.1: In Manual Mode, default signals SHALL be generated using
              potentiometer outputs
REQ-FR-004.2: Default signal values SHALL be:
              - Signal 1 (S1): 350 mV
              - Signal 2 (S2): 700 mV
REQ-FR-004.3: The system SHALL read potentiometer values when relay is active
REQ-FR-004.4: The system SHALL output potentiometer values to O1 and O2 outputs
              for measurement and calibration

2.5 Mode 2 - Automatic Operation (FR-005)
------------------------------------------
REQ-FR-005.1: In Automatic Mode, default signals SHALL be generated by ESP32
              using stored calibration values
REQ-FR-005.2: The system SHALL support calibration button functionality
REQ-FR-005.3: When calibration button (IO4) is pressed:
              - System SHALL read actual pedal signals (APS1, APS2)
              - System SHALL store these values as default signals
              - Calibration SHALL only occur when pedal is in zero position
REQ-FR-005.4: Calibrated values SHALL be stored in non-volatile memory
REQ-FR-005.5: The system SHALL regenerate stored default values when speed
              limit is exceeded

2.6 Signal Processing (FR-006)
--------------------------------
REQ-FR-006.1: The system SHALL read actual pedal signals from:
              - APS1: IO39 (ADC input)
              - APS2: IO36 (ADC input)
REQ-FR-006.2: Actual pedal signal readings SHALL be corrected using formula:
              Actual reading = ADC reading × (69.6 / 47.5)
REQ-FR-006.3: The system SHALL generate stimulated default pedal signals on:
              - SDPS1: IO21 (DAC/PWM output)
              - SDPS2: IO22 (DAC/PWM output)
REQ-FR-006.4: Output signal generation SHALL account for board voltage divider:
              To generate 1V output, system SHALL generate 2V internally

2.7 Relay Control (FR-007)
---------------------------
REQ-FR-007.1: The system SHALL control relay via IO23
REQ-FR-007.2: Relay SHALL be activated (HIGH) when speed limit is exceeded
REQ-FR-007.3: Relay SHALL be deactivated (LOW) when speed is below limit
REQ-FR-007.4: Relay control SHALL support external signal override capability

2.8 Warning System (FR-008)
----------------------------
REQ-FR-008.1: The system SHALL activate buzzer (IO14) when speed limit is
              exceeded
REQ-FR-008.2: Buzzer SHALL produce audible warning tone
REQ-FR-008.3: Buzzer SHALL deactivate when speed decreases below limit

2.9 Communication Interfaces (FR-009)
---------------------------------------
REQ-FR-009.1: The system SHALL support CAN bus communication:
              - CAN RX: IO32
              - CAN TX: IO33
              - Baud rate: Standard automotive CAN (500 kbps or 250 kbps)
REQ-FR-009.2: The system SHALL support RS232 communication:
              - RS232 TX: IO15
              - RS232 RX: IO34
              - Baud rate: Configurable (default: 115200)

2.10 Digital I/O (FR-010)
--------------------------
REQ-FR-010.1: The system SHALL support digital input on IO13 (Pull-up enabled)
REQ-FR-010.2: The system SHALL support digital output on IO18 (Pull-out)
REQ-FR-010.3: Digital I/O states SHALL be readable/writable via software

3. NON-FUNCTIONAL REQUIREMENTS
================================================================================

3.1 Performance Requirements (NFR-001)
----------------------------------------
REQ-NFR-001.1: Speed monitoring loop SHALL execute at minimum 10 Hz
REQ-NFR-001.2: Response time from speed limit exceed to relay activation
               SHALL be < 100 ms
REQ-NFR-001.3: ADC sampling rate for pedal signals SHALL be minimum 1 kHz
REQ-NFR-001.4: CAN message processing SHALL not block main control loop

3.2 Reliability Requirements (NFR-002)
---------------------------------------
REQ-NFR-002.1: System SHALL operate continuously without requiring restart
REQ-NFR-002.2: Calibration data SHALL persist across power cycles
REQ-NFR-002.3: System SHALL handle CAN bus communication errors gracefully
REQ-NFR-002.4: System SHALL default to safe state (relay OFF) on initialization

3.3 Safety Requirements (NFR-003)
-----------------------------------
REQ-NFR-003.1: System SHALL fail-safe: relay defaults to OFF (pedal connected)
REQ-NFR-003.2: System SHALL validate calibration data before use
REQ-NFR-003.3: System SHALL prevent invalid signal generation (out of range values)
REQ-NFR-003.4: Watchdog timer SHALL be implemented to detect system hangs

3.4 Usability Requirements (NFR-004)
--------------------------------------
REQ-NFR-004.1: Calibration process SHALL be simple (single button press)
REQ-NFR-004.2: System SHALL provide clear indication of current operating mode
REQ-NFR-004.3: Warning buzzer SHALL be clearly audible to driver

6. OPERATING MODES DETAILED SPECIFICATION
================================================================================

6.1 Mode 1: Manual Mode
------------------------
6.1.1 Initialization
- System SHALL detect manual mode via hardware switch
- System SHALL initialize potentiometer reading
- System SHALL set default signal values: S1=350mV, S2=700mV

6.1.2 Normal Operation Sequence
1. Monitor CAN bus for vehicle speed
2. When speed >= limit (40 km/h):
   a. Activate relay (IO23 = HIGH)
   b. Read potentiometer values
   c. Output potentiometer values to SDPS1 (IO21) and SDPS2 (IO22)
   d. Activate buzzer (IO14 = HIGH)
3. When speed < limit:
   a. Deactivate relay (IO23 = LOW)
   b. Stop outputting default signals
   c. Deactivate buzzer (IO14 = LOW)

6.1.3 Calibration Procedure
- Set hardware switch to manual mode
- Activate relay manually
- Measure O1 and O2 outputs
- Adjust potentiometers to achieve desired default values (350mV, 700mV)

6.2 Mode 2: Automatic Mode
---------------------------
6.2.1 Initialization
- System SHALL detect automatic mode via hardware switch
- System SHALL load stored calibration values from non-volatile memory
- If no calibration exists, system SHALL use default values

6.2.2 Calibration Procedure
1. Install board on target vehicle
2. Ensure pedal is in zero position (not pressed)
3. Press calibration button (IO4)
4. System SHALL:
   a. Read actual pedal signals (APS1, APS2)
   b. Apply correction formula: Actual = Reading × (69.6 / 47.5)
   c. Store corrected values in non-volatile memory
   d. Confirm calibration completion

6.2.3 Normal Operation Sequence
1. Monitor CAN bus for vehicle speed
2. When speed >= limit (40 km/h):
   a. Activate relay (IO23 = HIGH)
   b. Retrieve stored calibration values
   c. Generate signals accounting for voltage divider (2V for 1V output)
   d. Output to SDPS1 (IO21) and SDPS2 (IO22)
   e. Activate buzzer (IO14 = HIGH)
3. When speed < limit:
   a. Deactivate relay (IO23 = LOW)
   b. Stop outputting default signals
   c. Deactivate buzzer (IO14 = LOW)

7. DATA STORAGE REQUIREMENTS
================================================================================

7.1 Calibration Data (DS-001)
-------------------------------
REQ-DS-001.1: System SHALL store calibration values in non-volatile memory
REQ-DS-001.2: Stored data SHALL include:
              - APS1 calibrated value (float, 4 bytes)
              - APS2 calibrated value (float, 4 bytes)
              - Calibration timestamp (optional)
REQ-DS-001.3: Calibration data SHALL persist across power cycles
REQ-DS-001.4: System SHALL validate calibration data on startup

7.2 Configuration Data (DS-002)
--------------------------------
REQ-DS-002.1: System SHALL store configurable parameters:
              - Speed limit value (default: 40 km/h)
              - CAN message ID for speed reading
              - CAN speed calibration factor
              - RS232 baud rate

8. ERROR HANDLING AND DIAGNOSTICS
================================================================================

8.1 Error Conditions (EH-001)
-------------------------------
REQ-EH-001.1: System SHALL handle CAN bus communication errors
REQ-EH-001.2: System SHALL handle invalid speed readings
REQ-EH-001.3: System SHALL handle ADC reading failures
REQ-EH-001.4: System SHALL handle calibration data corruption

8.2 Error Responses (EH-002)
-----------------------------
REQ-EH-002.1: On CAN error, system SHALL maintain last known speed for
              maximum 5 seconds, then default to safe state
REQ-EH-002.2: On invalid calibration data, system SHALL use default values
              and indicate error via RS232
REQ-EH-002.3: On system hang, watchdog SHALL reset system to safe state

8.3 Diagnostic Output (EH-003)
--------------------------------
REQ-EH-003.1: System SHALL output diagnostic information via RS232
REQ-EH-003.2: Diagnostic information SHALL include:
              - Current speed reading
              - Current mode
              - Relay state
              - Calibration status
              - Error codes (if any)

10. CONSTRAINTS AND ASSUMPTIONS
================================================================================

10.1 Constraints
----------------
CON-001: System is designed for vehicles with electric wired brake pedals
CON-002: Target vehicle for testing: Nissan Sunny Car
CON-003: Speed limit for testing: 40 km/h (dashboard speed)
CON-004: CAN speed readings require calibration to match dashboard speed
CON-005: Output signals require 2:1 voltage divider compensation

10.2 Assumptions
-----------------
ASM-001: Vehicle CAN bus is accessible and provides speed information
ASM-002: Pedal signals are analog voltage signals (0-3.3V range)
ASM-003: ECU accepts analog voltage signals for pedal input
ASM-004: Hardware switch reliably indicates operating mode
ASM-005: Calibration button is pressed only when pedal is at zero position

11. GLOSSARY
================================================================================

APS1/APS2:      Actual Pedal Signal 1/2 - The real-time pedal position signals
CAN:            Controller Area Network - Automotive communication protocol
ECU:            Engine Control Unit - Vehicle's main computer
SDPS1/SDPS2:    Stimulated Default Pedal Signal 1/2 - Generated default signals
RS232:          Serial communication standard
ADC:            Analog-to-Digital Converter
DAC:            Digital-to-Analog Converter
PWM:            Pulse Width Modulation
IO:             Input/Output pin

12. APPENDICES

12.1 Signal Correction Formula
-------------------------------
Actual Pedal Signal = ADC Reading × (69.6 / 47.5)

12.2 Output Signal Generation
------------------------------
Required Output Voltage = Desired Voltage × 2
(Account for 2:1 voltage divider on board)

12.3 Default Signal Values (Mode 1)
------------------------------------
Signal 1 (S1): 350 mV
Signal 2 (S2): 700 mV
